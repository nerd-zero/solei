## Known Pain Points

**Deployment pipeline** — complex, slow, or hard to debug when it breaks.
- Multi-stage: build Docker image → push to GHCR → Render deploy (sandbox first, then production)
- deploy_server.sh polls Render API manually; timeouts and ordering are fragile
- Terraform manages service IDs — any mismatch between TF state and Render causes silent failures
- Vercel staging → promotion adds another async step

## Engineering Conventions

From CLAUDE.md and server/CLAUDE.md:

- **Repository pattern**: ALL database queries must be in `repository.py` files — never in services or endpoints
- **No `session.commit()`**: sessions auto-commit at end of request/task; use `session.flush()` if needed mid-request
- **Auth-aware queries**: always filter by `auth_subject` in repository `get_readable_statement()`
- **Background jobs**: use `enqueue_job()` — never call background logic inline
- **Read vs write sessions**: `AsyncReadSession` for queries, `AsyncSession` for mutations
- **Schemas**: read schemas include all fields; update schemas have all fields optional with `None` default
- **Module structure**: each module has endpoints / service / repository / schemas / tasks — no exceptions
